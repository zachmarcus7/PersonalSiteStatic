ArtLocal

ArtLocal

Dynamic website built with Angular, ASP.NET Core Web API, Entity Framework Core, Microsoft SQL Server, and TypeScript. Allows users to create new accounts, log in, and buy specific pieces of art. Also has an admin dashboard where admins can perform CRUD capabilities on Entities in a SQL database.


Angular, ASP.NET Core Web API, Entity Framework Core, Microsoft SQL Server, TypeScript
Close
Landing Page

This is the landing page for the website. Here, users can browse different pieces of artwork that are available. To view this section, a user doesn't need to be authenticated.

Customer Login Page

Once a customer logs in, their credentials are sent as a POST request to the API. Once they're authenticated, a JWT is generated and sent back, which gets stored in the user's local storage.

Customer Registration

With this page, a new Customer entity can be created and stored in the SQL Server with the API and Entity Framework Core. Once that's done, the customer will be authenticated, and their details will be stored in local storage.

Artwork Details Page

Once a user is logged in, their username gets displayed in the top right corner of the navigation bar, along with the option to log out. Also, when a specific artwork is selected, this page will be displayed. It makes two different calls to the API to get details about the a specific piece of Artwork and its corresponding Artist.

Checkout Page

Once the "Place Order" button is clicked on this page, Angular calls the API to update the Artwork as sold as well as to create an Invoice connecting the Customer to the Artwork.

Artwork Details Page Updated

Once a Customer places an order on a specific piece of Artwork, this page gets updated to let other Customers know that this specific piece of Artwork has become unavailable.

Admin Login Page

Just like with the Customer login page, a JWT is sent back once an admin is authenticated. However, when an admin authenticates, their role is set to "admin" inside the JWT, allowing them more privileges with the API.

Admin Login Page Validation

This is just an example of how the project used Angular Directives to validate login forms. So when an admin doesn't enter the correct credentials, the Directive gets executed and an error is displayed.

Admin Dashboard

This is one of the pages that the admin has access to inside the admin dashboard, which is protected by an Angular Auth Guard. With the admin dashboard, there's a nested navigation menu on the left where the admin can select different entities to update.

Artwork Data Page

This is another page inside the admin dashboard, where the admin can view details about specific Artwork entities.

Artwork Creation Page

With this page, several different API calls are used to generate dropdown selections for existing entities in the SQL database that the Artwork entity is connected to.

Gallery Creation Page

This is another page inside the admin dashboard where the admin can create a new Gallery entity in the database.
